# מסמך אפיון מערכת - iris-demo-site

גרסה: 0.1  
תאריך: 2026-04-15  
סטטוס: טיוטת אפיון ראשונית

## 1. מטרת המסמך

מסמך זה מגדיר אפיון ראשוני למערכת **iris-demo-site**.  
מטרת המסמך היא לייצר בסיס מוסכם לפיתוח, לארגון הקוד, לחיבור מול GitHub/Codex, ולהמשך הרחבת המערכת באופן מבוקר.

המסמך מבוסס על:
- מבנה הפרויקט שהוגדר עד כה.
- ההקשר הכללי של הפרויקט כפי שעלה בשיחה.
- הנחות עבודה סבירות שנדרשו לצורך גיבוש אפיון ראשוני.

במקומות שבהם לא הוגדרה דרישה חד-משמעית, הדבר מצוין במפורש כהנחת עבודה.

---

## 2. רקע והקשר

המערכת מיועדת לשמש כאתר הדגמה / סביבת דמו יישומית בשם **iris-demo-site**.

מטרות-העל של המערכת הן:
- להציג יישום web מסודר, קל להרצה וקל להרחבה.
- לייצר בסיס קוד ברור עבור עבודה עם GitHub ו-Codex.
- לאפשר הפרדה טובה בין שכבות המערכת: ניתוב, שירותים, מודלים, תבניות ותוכן סטטי.
- לאפשר המשך פיתוח הדרגתי תוך שמירה על סדר, תיעוד ויכולת תחזוקה.

## 6. הנחות עבודה

הסעיפים הבאים הם **הנחות עבודה** ולא החלטות מחייבות, עד לאישור מפורש:

1. המערכת מבוססת Python.
2. המערכת רצה כאפליקציית Web בצד שרת.
3. הקובץ `app.py` משמש כנקודת הכניסה הראשית.
4. המערכת משתמשת במבנה Templates + Static קלאסי.
5. קיימת חלוקה לשכבות `routes`, `services`, `model`.
6. הפרויקט מיועד לעבוד היטב עם GitHub ועם סיוע של כלי AI.
7. בדיקות ייכתבו ב-`unittest`.
8. נדרש בסיס טוב להמשך אינטגרציה עתידית, אך לא בהכרח למימוש מלא כעת.

## 7. תיאור כללי של המערכת

המערכת בנויה כיישום Web מודולרי בעל מבנה שכבות ברור:

- **שכבת כניסה** - `app.py` אחראית לאתחול האפליקציה ולהרשמת הרכיבים הראשיים.
- **שכבת ניתוב** - `routes/` מרכזת את הגדרות הנתיבים והבקרים.
- **שכבת שירותים** - `services/` מכילה לוגיקה שאינה קשורה ישירות ל-HTTP או לתצוגה.
- **שכבת מודלים** - `model/` מגדירה מבני נתונים, DTOs או מחלקות דומיין.
- **שכבת תצוגה** - `templates/` מכילה קבצי HTML, ו-`static/` מכילה CSS, JS ונכסים סטטיים.
- **שכבת בדיקות** - `tests/` כוללת בדיקות יחידה ואולי גם בדיקות אינטגרציה בסיסיות.
- **שכבת תיעוד** - `docs/` מכילה מסמכים, הסברים ותיעוד טכני.

## 9. מבנה תיקיות וקבצים

להלן מבנה צפוי ברמת על:

```text
iris-demo-site/
├── .github/
├── .vscode/
├── docs/
├── model/
├── routes/
├── services/
├── static/
├── templates/
├── tests/
├── .gitignore
├── AGENTS.md
├── Dockerfile
├── TODO.md
├── app.py
└── requirements.txt
```

### 9.1 פירוט ייעוד התיקיות

#### `.github/`
קבצי workflow, הנחיות תהליך, אוטומציות CI/CD עתידיות והגדרות הקשורות ל-GitHub.

#### `.vscode/`
הגדרות סביבת עבודה מקומית למפתחים המשתמשים ב-VS Code.

#### `docs/`
תיעוד פונקציונלי, טכני, ארכיטקטוני ותפעולי.

#### `routes/`
מימוש endpoints, חלוקת אזורי האתר, וניתוב בקשות.

#### `services/`
לוגיקה עסקית, פעולות עיבוד, תקשורת לשירותים חיצוניים, עזרי מערכת.

#### `model/`
מחלקות נתונים, אובייקטי דומיין, סכמות וייצוגים פנימיים.

#### `templates/`
קבצי HTML מבוססי template engine.

#### `static/`
CSS, JavaScript, תמונות, אייקונים ושאר קבצים סטטיים.

#### `tests/`
בדיקות יחידה, בדיקות אינטגרציה בסיסיות וציוד בדיקה.
